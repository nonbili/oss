Nora prend en charge le CSS personnalisé afin que vous puissiez remodeler les sites web des services qu'il intègre.

## Utilisations courantes

- Masquer l'interface utilisateur distrayante
- Améliorer la densité pour les mises en page riches en fils d'actualité
- Supprimer les éléments spécifiques au service que vous n'utilisez jamais
- Faciliter le balayage visuel des configurations multi-colonnes sur bureau

## Garder les changements maintenables

Comme Nora dépend des sites web en direct de services externes, le balisage source peut changer sans préavis.

Utilisez une approche conservatrice :

- Préférez des règles petites et ciblées
- Testez à nouveau après les refontes de services
- Évitez les remplacements globaux à moins de maîtriser les conséquences

## Associer le CSS aux profils

Si vous utilisez des profils séparés pour différents services ou rôles, gardez les ajustements CSS alignés avec ces profils pour que le comportement reste prévisible.
